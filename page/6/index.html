什么是字符设备

左边的这幅图片中的键盘和串口就是很典型的两个字符设备。

在 Linux 中， 字符设备是驱动程序通过发送或者接收字符与之通信的设备。通过字符设备可以直接访问到硬件，不需要任何缓冲。
如果站在用户空间来看字符设备，其是一个文件（相信大家应该都知道了，在Linux的世界里，一切皆文件），如果站在内核空间来看字符设备，其就是一个结构体。
这个你们先听一下了解一下，后面会讲到

char driver overview

接下来，我们来看一下Linux中的字符设备驱动具体是什么样子的。

字符驱动主要是kernel driver module这一部分， 驱动里面我们会实现一些操作字符设备的函数，譬如 open， read， write。这些函数就对应与我们用户空间文件操作函数

从这幅图我们也能看到字符设备的整个工作流程，有几个关键的部分：硬件，驱动，VFS，应用程序。VFS 是 Virtual File system 的简写，也就是虚拟文件系统，其在内核空间，主要给用户空间的应用程序提供（更准确点讲是不同的用户空间文件系统，比如ext4，这个你们现在了解一下就可以了）接口，这样子用户空间就不用关注底层实现，调用 write 函数就会直接写入内核中的字符设备，并最终反应到硬件。

首先开机的时候或者说挂载驱动的时候， driver 会在 VFS 中去注册字符设备。 然后也会通过 class_create 和 device_create 等去创建应用空间可以访问的文件（或者说设备节点）。完成这些动作后用户空间就可以通过系统调用访问字符设备了。

The Device number

用户空间和内核怎么取得联系的呢？ 答案就是 device number， device number 分为major number 和 minor number

major number 表明这个设备使用的哪一份 driver， 0，1，2 … 代表 instance 设备实例，在Linux 中 driver 和 device 的关系有点向面向对象编程语言里面的类与实例， 一个 driver 可以对应多个 设备。（这句话，我觉得你们可以记录一下，因为我没有写到 PPT）

比如我们有一个 rtc 设备，注册驱动的时候 major 分配为 5. minor 为 0 ， 那么在用户空间就可以看到设备号。主 5 次 0

假如我有多个 rtc 设备， 大家觉得设备号会是怎么分配呢？ 有谁愿意来讲一下吗

对，不管多少设备，只要是同类型的，都可以使用同一份驱动，只是会有多个设备实例

比如说我们可以看下系统中的设备文件： c 开头表明是字符设备 tty

file_operations 比较重要

symbols:
存在的目的就是让外部软件可以知道kernel文件内部实际分配的位置

Unix 明确区分文件内容和文件信息
inode 是一种 VFS 数据结构（struct inode），用于保存有关文件的一般信息。
VFS“文件”数据结构（结构文件）跟踪用户进程对打开文件的交互
Inode 包含文件系统处理文件所需的所有信息。
每个文件都有自己的 inode 对象，文件系统使用它来标识文件
每个 inode 对象都与一个 inode 编号相关联，该编号唯一标识文件系统中的文件。
在创建新文件（常规文件或设备）时，将创建 inode 对象并将其存储在内存中。

inode_operations 先大概了解下，感受下它是做什么的就可以了，等熟悉一些再去深入研究

每当打开文件时，都会在内核空间中创建一个文件对象。 每次打开常规文件或设备文件都会有一个文件对象。
存储有关打开文件和进程之间交互的信息
此信息仅在进程打开文件期间存在于内核内存中。 与 inode 不同，文件对象的内容不会写回磁盘。
